## 目录结构

（1）`Tree` 树结构分析

**在开始之前，您务必了解它来提升开发速度，避免猜测。**

```
|-- stack
        |-- common   // 可能会保存一些公共类
        |-- config 		// 目前只有数据库的配置，您可以在这里添加各项配置文件，此文件夹目的就是为了保存所有的配置文件
        |-- dist  // 使用Typescript的生成目录
        |-- docs   // 保存该项目的文档
       	|-- model // 该目录保存项目的数据库model对应实体对象
       	   ||-- db.ts  // db连接池文件
       	   ||-- entity // model实体对象，定义Schema在这里
       	   		||-- users.model.ts
       	   		||-- ...
       	   ||-- types  // 定义类型
       	      ||-- users.ts // export interface IUser{}
       	      ||-- ...
       	||-- routers //所有的controller在这里
       		 ||-- users.controller.ts
       		 ||-- ...
        |-- service // service文件夹
           ||-- user.service.ts
           ||-- ...
        |-- test // 您可以在这里写测试用例
           ||-- userservice.test.ts
           ||-- ...
        |-- tools // 需要用到的工具代码放在该文件夹下
        |-- types // 定义HTTP API的参数，参数类型，返回值，返回类型
        	 ||-- api.users.ts
        |-- app.ts // 程序运行主文件
        |-- constant.ts  // 可能会用到的配置——生成token所用
        |-- jwt.ts // 您可能需要排除某些API使用token，那就设置这个文件
        
        ... // 可能剩余通用配置文件，如git排除文件或者是tslint文件等等
```
